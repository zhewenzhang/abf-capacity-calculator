# 多 Agent 协作系统

## 调用方式
`/multi-agent <任务描述>`

## 功能说明
将复杂任务自动分解为子任务，调度合适的专业 Agent 并行或串行执行，最终整合为完整的解决方案。

## 可用 Agent 团队

| Agent | 角色 | 专长 |
|-------|------|------|
| **Orchestrator** | 总控调度 | 任务分解、进度追踪、结果整合 |
| **Researcher** | 代码研究 | 代码库分析、架构理解、模式识别 |
| **Implementer** | 代码实现 | 编写高质量生产代码 |
| **Reviewer** | 代码审查 | 质量检查、安全审计、性能分析 |
| **Tester** | 测试验证 | 测试设计、用例编写、结果验证 |

## 标准工作流

```
用户输入
    ↓
Orchestrator: 任务分析与分解
    ↓
    ├─→ Researcher: 代码研究（并行）
    ├─→ Researcher: 影响分析（并行）
    ↓
Orchestrator: 制定实现计划
    ↓
Implementer: 代码实现
    ↓
    ├─→ Reviewer: 代码审查（并行）
    ├─→ Tester: 测试验证（并行）
    ↓
Orchestrator: 结果整合与质量检查
    ↓
最终输出
```

## 使用示例

```
/multi-agent 为项目添加一个新的用户权限管理功能，
包含角色定义、权限检查中间件、UI 权限控制组件。
需要确保类型安全和完整的测试覆盖。
```

## 输出内容
- 任务分解报告
- 每个子任务的执行结果
- 整合后的完整解决方案
- 质量检查报告
- 后续改进建议
