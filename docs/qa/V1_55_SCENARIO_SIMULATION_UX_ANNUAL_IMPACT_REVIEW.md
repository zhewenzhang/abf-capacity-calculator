### 2. 年度倍率情境计算校验 (检查项 2)
* **审计状态**：`✅ PASS`
* **详细发现**：
  * `scenarioEngine.ts` 中新增了 `applyAnnualMultipliers`，按月提取年份 `yearOf(month)` 完美套用 `annual` 字典中的年度倍率。预测数量 (`forecastVolume`)、单价倍率 (`unitPrice`)、Core 产能倍率 (`coreCapacity`)、BU 产能倍率 (`buCapacity`) 全部在年份上生效，从底层打通了年度情境。
  * 全局倍率输入逻辑与转换正确，输入百分比（如 `+10%` / `-15%`）通过 `multiplierFromPct` 转化为正确的倍率乘数（`1.10` / `0.85`）并一键应用至所有年份，每年对应的格子及计算结果均自动更新。
  * 全局批量套用后，用户在矩阵表格中对任意年份的格子进行修改，会通过 `handleCellChange` 重新写入特定年份的 multipliers 字典中，微调逻辑完美覆盖全局值。
  * 矩阵侧边的「新增前一年 / 后一年」按钮通过 `handleAddYear` 触发，向 multipliers 写入默认倍率，并自动按照年份排序在表格中渲染。

### 3. 三模式结果表重构 (检查项 3)
* **审计状态**：`✅ PASS`
* **详细发现**：
  * 彻底废除了原先“基线 / 情境 / 变化”混成横向大表的累赘结构。改为了使用 `<Segmented>` 模式切换器的精简表格，列结构仅保留：`指标 + 年份`。
  * 保留的指标均为高价值核心指标：总营收、BP 达成率、BP 差距、总预测数量、最大 Core 利用率、最大 BU 利用率、短缺月份。
  * 原始模式 (`original`)、模拟模式 (`simulated`) 以及变化模式 (`delta`) 的数据口径符合业务规范：
    * **营收**：在变化模式下展示金额变化（`+X.XM` 或 `-X.XM`）。
    * **BP 达成率**：在变化模式下展示百分点变化（`+X.Xpp` 或 `-X.Xpp`）。
    * **BP 差距**：在变化模式下展示金额变化（`+X.XM`）。
    * **利用率**：在变化模式下展示百分点变化（`+X.Xpp`）。
    * **短缺月份**：在变化模式下展示月份差值（`+X`）。

### 4. 摘要 KPI 与阈值警示卡片 (检查项 4)
* **审计状态**：`✅ PASS`
* **详细发现**：
