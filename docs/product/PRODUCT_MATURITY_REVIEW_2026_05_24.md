# ABF 产能计算器产品成熟度诊断审查报告 (Product Maturity Review)

- **评估时点**：2026 年 5 月 24 日
- **系统版本基准**：`v1.24.1`

ABF 产能计算器旨在解决半导体 ABF 载板复杂多变的销售预测、良率波动、各制程产能负载（Bottleneck bottleneck）以及年度目标（BP）缺口核算的复杂运筹系统。本报告对当前计算器的技术现状、商业成熟度与演进路径执行全方位的工程大体检。

1. **多币别与汇率结算 (4.2分 - 优秀)**：具备了 SKU 多币种输入、汇率自动归一化美元对账机制，但在历史汇率快照追踪层面仍有进步空间。
2. **BP Targets 营业目标分析 (3.8分 - 良好)**：年度目标达成率核算逻辑完备，但旧版 Parameters 入口并存导致的并发覆写冲突是目前最大体验隐患（v1.25 即将彻底物理断开）。

5. **AI Brief Export (3.5分 - 良好)**：集成了安全导流机制，防止 AI 归因结论误导，但目前仍偏向纯文本导出，缺少深度决策模拟交互。

## 2. 四大软件工程成熟度维度诊断 (Software Engineering Assessment)

### 📊 A. UI / UX 视觉与操作成熟度 (评级：B+ / 良好)
- **现状**：全局采用 Ant Design + 渐进式 Glassmorphism 样式，主视觉和谐。
- **最大缺口**：
  - 高频电子表格录入场景在老版本中采用了散装的 React 表单输入，缺乏流畅的键盘贴附导航体验。
- **优化路径**：
  - 通过 v1.25.0 （BP targets react-datasheet-grid 独立页）及 v1.26.0（Forecasts 12个月横向铺设大表）灰度并行重塑高频录入板块，完成表格视觉 Parity。

### 💾 B. 数据治理与防灾成熟度 (评级：B / 良好)
- **现状**：Firestore 实现了多维嵌套的数据读取，且前端做了一定程度的 memo 缓存。
- **最大缺口**：
  - 缺乏对 `null`（未设置）与 `0`（数字零）的严密语义化隔离，导致 Attainment（达成率）分母存在除零白屏崩溃的边缘隐患。
  - **Firestore 成本控制隐患**：若后期导入海量 SKU 录入，失焦自动保存会引发巨额物理写入计费。
- **优化路径**：
  - 强制禁止 Excel-like 大表的 Auto-save 行为，统一通过公共 Dirty 状态机实施 writeBatch() 批量极简 Patch 保存。

### 🔐 C. 协同与权限安全成熟度 (评级：B- / 中等)
- **现状**：实现了 Personal、Owner、Editor 和 Viewer 四级角色的权限划分。
- **最大缺口**：
  - **前端 Viewer 漏防**：部分 Excel 录入实验表虽然禁用了保存，但未在网格级设置只读 `readOnly` 限制，Viewer 依然能双击编辑引发虚假数据已存的误解。
- **优化路径**：
  - 完善 DOM 级硬只读属性逻辑，并将状态条封装为 `DirtyStateActionBar` 全局守卫。

### 🤖 D. AI 智能决策与科学归因成熟度 (评级：C+ / 一般)
- **现状**：支持将对比差异打包为 Brief 提示词，并引入了安全出口防线。
- **最大缺口**：
  - 存在严重的数学概念表述不清：把 top changes 产生的比例分摊归因（数学分母占比，Proportional Attribution）在 UI 措辞上说成了是引发缺口的物理因果关系（Causality），易误导高管做出错误投资。
- **优化路径**：
  - 彻底治理全站中英文翻译包，将“因……导致”换装为中性的“占比贡献度”。

3. **SLA 性能防线跑通**：在 100 SKU 与 500 SKU 场景下跑通首屏 FP 渲染和粘贴帧数限制，确保网格不粘滞。

### 🔮 未来 3 个月商业化构想
- **Month 1**：全站录入状态机大合并（v1.28.0），完成从“实验 Lab”向“生产级大表”的平滑灰度转正。
- **Month 2**：自研 **TanStack Table + Virtual Headless** 二维网格引擎，支持 10,000+ 级超大型 SKU 销售/良率产能矩阵的高性能流畅修改与复制粘贴。
- **Month 3**：引入 **AI 智能多维决策沙盘 (Sandbox)**。高管可在系统内进行 “汇率波动 +/-5%”、“某良率下降 10%” 场景的蒙特卡洛预测模拟，将纯文本导出飞跃为真正智能的运筹看板。

## 🏁 跃升为“商业化成熟产品”的 4 大硬性定义线

当且仅当系统跨过以下四条硬性工程分界线时，方可被确认为一款高可信度、完全成熟的商业级 SaaS 软件：

1. ⚡ **性能 SLA 铁律**：在单工作空间包含 **500个以上物理 SKU** 且 2 年滚动周期场景下，表格首屏加载完全呈现在 **1.5秒之内**，单单元格打字延迟在 **60ms以内**，千级粘贴无挂起。
2. 🔒 **多租户数据零越权安全**：本地 Firestore Emulator 安全用例集 100% 覆盖 Viewer/Editor/Owner，在只读状态下 100% 拦截任何直接通过终端物理 API 的非法写绕过。
3. 💰 **资费红线控制**：运行一整天高频录入，由于引入了 Batch Save 对比 Patch 策略，单用户单日 Firestore 产生 write 计费物理写入次数绝对不超过 **50 次**。
4. 📈 **财务级对齐精度**：多币种销售额标准化美元再转台币折算逻辑无懈可击，与财务系统、外部物理 Excel 进行多周期累计对账时，两端数据达到 **0.00% 零分差对账精度**。
