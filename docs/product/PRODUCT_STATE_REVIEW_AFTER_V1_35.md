# v1.35.0 之后 ABF 产能规划系统产品状态回顾 (Product State Review)

随着 `v1.35.0`（Data Quality Visibility 数据质量前移）版本的成功发布，本系统已完成了前端基建、设计系统、基本业务要素录入以及确定性分析引擎的全面拼图。在此，我们对当前的产品状态、核心能力以及产品定位进行深度复盘。

---

## 一、 目前产品做到什么程度 (Product Capabilities Assessment)

截至 `v1.35.0`，系统已稳健构建了以下八大核心能力板块，具备了极高的完备度与业务逻辑闭环度：

1. **SKU / Forecast / Capacity / BP Targets (基础数据要素管理)**：
   - **SKU 管理**：支持产品 SKU（包括芯片尺寸、层数、层数分类、良率、单价、计价币别）的全面 CRUD 管理。
   - **销售预测 (Forecast)**：支持长达 15 年（2026–2040）的月度销售预测录入。既有符合正式流程的 AntD 列表视图，也提供了实验室版本（Forecasts Spreadsheet Lab）支持高效 Excel 复制粘贴的 Spreadsheet 交互。
   - **产能规划 (Capacity)**：支持多工厂、多年月维度的 Core/BU 产能配置，具备 Fill Forward 极速填充以及年月聚合视图。
   - **营业目标 (BP Targets)**：支持独立的营业目标录入，采用 Excel-like 的横向时间序列管理，与产能和收入无缝挂钩。

2. **多币种支持 (Multi-currency)**：
   - 系统原生支持 USD 与 TWD 的双向无缝切换。
   - 允许用户在参数配置（Parameters）中设置 constant（常数）或 yearly（年度变动）汇率，所有收入预测和 BP Gap 计算均能实现毫秒级的币种转换与趋势更新。

3. **多人协同工作区 (Workspace collaboration)**：
   - 实现了基于 Firebase Firestore 的多人共享工作区机制。
   - 严密的权限模型设计：划分了 `Owner`（所有权）、`Editor`（编辑权）、`Viewer`（只读权）三个角色。
   - 针对 Viewer 角色，所有页面和 Spreadsheet 单元格均具备强置灰与物理写入拦截，保障数据资产的绝对安全。

4. **版本快照与变更评审工作流 (Snapshot & Change Review)**：
   - **版本快照**：支持保存、还原和删除带有多元元数据（版本类型 Tags 如 BP Baseline/Scenario 等、评审状态 Review Status、版本备注）的命名快照。
   - **变更评审 (Change Review)**：支持任意两个版本快照的横向变更比对，明晰单价（Price Impact）与需求数量（Quantity Impact）引起的营收变化归因，提供精准的瀑布流式归因分析。

5. **确定性风险简报 (Risk Brief)**：
   - 彻底摆脱对随机性 AI 接口的依赖，完全基于确定性数理公式运行。
   - 自动诊断出高风险压力警示期、WPI（加权产能压力指数）瓶颈，并从 Sales（销售）、Planning（生产规划）、Finance（财务）三大角色的视角自动生成决策级关注提示。

6. **无污染 AI 简报导出 (AI Brief Export)**：
   - 专为企业信息安全设计。一键生成 100% 脱敏（去除了敏感客户名称与具体工厂代号）的 JSON 载荷。
   - 搭配内置的 FAIR 诊断框架提示词（支持 EN/zh-TW 双语），安全引导用户在本地剪贴板复制，并粘贴至企业信任的本地或外部大模型（如 DeepSeek/Claude）进行深度的策略咨询，杜绝机密外泄。

7. **一致性 UI 系统 (UI System)**：
   - 经历 v1.30 - v1.34 多个版本的视觉规范洗礼，全局建立了一致的 `abf-page` 容器、`<PageHeader>`、`<ActionBar>`、`<UnitText>` 等组件。
   - 统一了数字格式化工具（Thousand Separators 千分位、百分比 1 位小数、NaN 优雅回退为 '-' 等），消除了数据呈现上的杂乱与不一致。

8. **数据质量前移可视化 (Data Quality Visibility)**：
   - 实践“左移 (Shift-Left)”诊断原则，在 Products、Forecasts、Capacity、BP Targets、Parameters 五大输入页面实现 inline 级的数据缺陷警示。
   - 自动检测并提示：缺失核心属性、孤儿预测、产能未配、目标脱节、汇率缺失等 DQ 问题，并赋予其 `decisionImpact` 严重级标签，确保源头数据的“高可信度”。

## 二、 产品的核心定位判断 (Product Positioning)

在经历了一系列迭代后，我们需要以第一性原理重新回答：“这个产品现在最像什么？”

### 候选定位深度剖析：
- **选项 A：Calculator (计算器)** ❌ — 过于局限。计算器属于单向计算工具。本系统已具备多用户协同、版本库管理、多币种动态转换、以及精密的快照变更评审归因流，远远超越了单纯公式运算的工具范畴。
- **选项 B：Planning dashboard (规划看板)** ❌ — 不够准确。看板（Dashboard）本质上是被动、只读的数据可视化媒介。而本系统强调深度的数据双向交互、版本演进以及主动式的缺陷诊断。
- **选项 C：Lightweight SaaS (轻量级 SaaS)** ❌ — 这仅是工程与部署形态，而非产品核心的业务属性。它回答了“代码怎么跑”，却没有回答“用户用它来创造什么价值”。
- **选项 D：Decision Analysis Tool (协同决策分析工具)** ✅ — **这是最精准的产品定位判断！**

### 最终产品定位宣告：
> **本系统是一个“基于多人协同、版本控制、具有数据质量自诊断能力的轻量级基板产能规划与决策分析工具 (Collaborative Decision Analysis Tool)”。**

**判定理由**：
ABF 载板产能配置的核心矛盾是“高额的固定资产投资”与“剧烈波动的市场需求”之间的博弈。决策者（销售副总、运营总监、CFO）使用本系统的本质目的不是“录入数据”或“看图表”，而是为了**“评估不确定性，并做出最优的产能扩张与商业报价决策”**。
系统通过多人协同（Collaboration）确保多部门数据录入效率，通过版本对比（Change Review）厘清市场变化影响，通过 AI Export 隔离安全与本地高级洞察，通过 DQ Visibility 确保输入源头的“高信任度”。它的所有功能，均围绕着**“帮助决策者在准确的数据基础上，做出理性的规划决策”**这一核心主线展开。

## 三、 目前系统的最大缺口分析 (Core Gaps)

在定位明确为“协同决策分析工具”后，我们审视目前 v1.35.0 之后的系统，发现以下几个主要痛点和功能缺口限制了其向“企业工业级”的迈进：

1. **多场景沙盒仿真能力缺失 (Scenario Planning) ── 【致命缺口】**
   - **痛点描述**：目前用户所有的修改都是在“正式工作区（Working Branch）”进行，修改会立即写入 Firestore 并实时同步给其他协同用户。这导致用户**完全不敢**在系统内尝试 What-If 假设（例如：如果大客砍单 15% 会怎样？如果某工厂良率提升 3% 会怎样？）。用户目前只能被迫导出 Excel 在本地玩沙盒模拟，算好之后再录入回系统，导致系统的“决策分析”价值大打折扣。
   - **判断**：**这是目前阻碍产品爆发的核心痛点，必须首要解决。**

2. **数据质量缺陷仅能“看见”无法“一键修复” (DQ Remediation) ── 【中度缺口】**
   - **痛点描述**：v1.35.0 实现了极佳的 DQ Visibility，用户能在输入页面看到红色警示 Badge。但当发现错误时，用户必须手动跳转或在庞大的表格中定位该行进行修改，缺乏“就地一键自愈”的流畅体验。
   - **判断**：属于用户体验（UX）的闭环问题，能够显著提升操作效率。

3. **缺乏严格的版本审批与锁定机制 (Version Workflow 2.0) ── 【轻度缺口】**
   - **痛点描述**：目前任何人（包括 Editor 角色）都可以删除或篡改 Snapshot，且没有“锁定（Freeze）”机制。对于需要作为年度财务基线的版本，缺乏防止意外篡改的防线。
   - **判断**：对于大型企业客户非常关键，但目前可通过团队协作约定暂时规避。

4. **Firebase 安全规则缺乏自动化回归测试 (Emulator Rules Tests) ── 【技术债缺口】**
   - **痛点描述**：Firestore 权限规则（firestore.rules）越来越复杂，涉及工作区隔离、角色读写限制等。目前完全依赖人工验证，修改安全规则时存在极高的“权限越权”或“误伤只读”的崩溃隐患。
   - **判断**：属于系统稳定性的隐患，是急需补齐的安全工程底座。

5. **AI 简报本地化集成度低 (AI Brief v2) ── 【泡沫与红线冲突】**
   - **痛点描述**：目前 AI 导出仍需用户手动复制粘贴到外部网页，交互不够顺滑。
   - **判断**：由于企业对商业机密的红线限制，任何直接调用外部大模型 API 的方案在合规上都难以通过；而在前端集成 WebLLM 又会带来极大的内存与加载负担。此功能目前属于高技术风险、低确定性收益的范畴。

6. **移动端/响应式适配体验不佳 (Mobile/Responsive Polish) ── 【低优缺口】**
   - **痛点描述**：大屏表格在手机等移动设备上的呈现和编辑体验较差。
   - **判断**：产能规划和决策分析属于典型的大屏桌面端高密度操作场景，移动端适配优先级极低。
