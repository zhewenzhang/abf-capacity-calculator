# 核心产品决策：为什么优先开发多场景沙盒仿真 (Scenario Planning) 而非 AI 简报二代 (AI Brief v2)？

在系统完成 `v1.35.0` 数据质量 Visibility 标准化后，产品团队针对“下一阶段最大亮点”展开了激烈讨论。备选核心方向主要为：
- **方案 A：AI 简报二代 (AI Brief v2)** — 进一步优化 AI 导出 payload，甚至在前端集成 WebLLM 或外部 API 直接在本机生成文字简报。
- **方案 B：多场景沙盒仿真 (Scenario Planning)** — 引入前端 Sandbox 物理隔离机制，让用户自由创建、篡改并横向对比 What-If 规划情境。

基于 **第一性原理 (First Principles Thinking)** 与 **KISS 简洁原则**，产品委员会做出最终决策：**无限期推迟 AI Brief v2，100% 优先启动 Scenario Planning！**

以下为深度决策依据与产品剖析：

---

## 一、 第一性原理：产业决策者真正的核心痛点是什么？

在 ABF 载板（Ajinomoto Build-up Film）产业链中，销售经理与工厂生产主管的核心痛点是：
> **“如果面临突发的外部市场波动，我该如何迅速、准确地评估不同的应对策略，并拿出一份具有确定性财务依据的仿真对比报告向高层汇报？”**

例如：
1. **Scenario 1**：大客苹果（Apple）突发要求降价 8%，且单月下单片数缩减 12%，我如果同意，全年的 BP 营收达成率会受多大影响？
2. **Scenario 2**：若财务要求今年底必须达成 1200M TWD 的营收线，我应该去协调哪家工厂把 Core 产能多扩充 10%，还是去游说客户多下哪款高层数 SKU 的单？
3. **Scenario 3**：如果我们将某条良率线提升 2%，这能换回多少额外的 Panel 产能释放？

### 现状痛点：
目前，用户在正式系统中**根本不敢**随意更改这些数据来做尝试。因为任何修改都会被直接同步到 Firestore 正式工作区，甚至直接触发其他协同在线用户的 Dashboard 和 Results 的实时重新计算，造成“规划数据雪崩”。因此，用户目前只能被迫将数据导出到本地 Excel 玩 What-If 计算，算好之后再录入回系统，系统的“决策分析”属性名存实亡。

---

## 二、 确定性价值 vs 随机性泡沫 (Scenario vs AI)

| 评估维度 | 方案 B：多场景沙盒仿真 (Scenario Planning) | 方案 A：AI 简报二代 (AI Brief v2) |
|---|---|---|
| **核心输出属性** | **确定性 (Deterministic) 的数理运算**。提供百分之百精确的 WPI 波动、BP 达成率、瓶颈月份对比。 | **随机性 (Probabilistic) 的文字摘要**。LLM 生成的分析具有一定的不确定性与幻觉概率。 |
| **商业决策信任度** | **极高**。工业规划决策者只信任 100% 精确的公式结果，不信任 AI 模糊的推理。 | **中等**。AI 生成的报告只能作为参考，无法直接用于财务和生产汇报。 |
| **合规与商业机密红线** | **0 风险**。所有沙盒数据、情境仿真 100% 运行在本地或工作区 Firestore 隔离子集内，不向外网泄露任何客户代码或产能密码。 | **高风险**。即使是 sanitized JSON，只要有 API 调用，就无法彻底说服企业法务相信其不会用于大模型再训练。 |
| **技术实现的 KISS 度** | **极简且扎实**。复用现存成熟的 `runCalculation`、`changeImpact` 比对框架，仅需设计轻量的前端 Sandbox 克隆机制。 | **极其繁重**。需解决 WebLLM 极其庞大的浏览器内存占用 (3G+ model)、长加载时间，或面临外部 API 调用的网络连通性。 |

---

## 三、 KISS 原则与开发复用最大化

从工程架构上评估，**Scenario Planning 能将我们前期沉淀的核心算法红利压榨出最大商业价值**：
- 我们的 `calculationEngine.ts` 是 pure stateless 的，能极其迅速地对任何输入结构进行毫秒级重算。
- 我们的 `changeImpact.ts` 引擎是通用的，只需将 `Base = 正式工作区版本`，`Target = 篡改后的沙盒情境` 丢入，就能瞬间产出极其震撼的“营收/BP 差距归因分析”和“单价 vs 数量敏感度”。

我们几乎不需要写任何复杂的第三方底层集成，只需在前台 UI 加上一个“**工作区 (Production) / 沙盒情境 (Sandbox Scenarios)**”的顶栏切换抽屉，就能瞬间将产品定位拔高到一个全新的高度。

## 四、 决策结论

因此，**多场景沙盒仿真 (Scenario Planning)** 具有极其清晰的商业场景、100% 确定性的决策说服力、0 商业数据泄露风险，以及极致的 KISS 工程实现度。我们判定，这是 v1.35.0 之后，ABF 产能规划系统迈向大企业市场所必须攻克的**一号核心阵地**。
