# ABF Capacity Calculator — 用户访谈脚本

**版本**: v1.0
**日期**: 2026-05-29
**用途**: 验证商业机会的用户访谈指南

---

## 一、访谈目标

### 核心假设

1. **痛点假设**: ABF 载板产能规划者目前用 Excel 做 What-if 分析，效率低、版本混乱、协作困难
2. **价值假设**: 一个集成了数据质量检测、场景模拟、AI 诊断的工具可以显著提升决策效率
3. **付费假设**: 目标用户愿意为"减少决策时间"和"避免产能误判"付费

### 验证指标

| 假设 | 验证标准 | 成功信号 |
|------|---------|---------|
| 痛点存在 | 70%+ 受访者确认 | "我每个月花 X 小时在 Excel 上" |
| 价值认可 | 60%+ 受访者认为有用 | "这个功能我马上就能用" |
| 付费意愿 | 30%+ 受访者愿意试用 | "我可以试试看" |

## 二、15 个核心问题

### A. 当前工作流程 (3 题)

**Q1: 请描述一下你每个月做产能规划的典型工作流程。从哪里开始？用什么工具？花多长时间？**

> 目标: 了解当前工作流程的复杂度和耗时
> 强信号: "我每个月花 2-3 天在 Excel 上"
> 弱信号: "还好，不太花时间"

**Q2: 你的产能数据、销售预测、BP 目标分别在哪里管理？这些数据怎么整合到一起？**

> 目标: 了解数据分散程度
> 强信号: "我有 5 个 Excel 文件，每次要手动复制粘贴"
> 弱信号: "都在一个系统里"

**Q3: 当你需要做一个 What-if 分析（比如"如果客户砍单 20%"），你现在怎么做？需要多长时间？**

> 目标: 了解 What-if 分析的痛点
> 强信号: "我做一个场景要半天，而且不敢改太多假设"
> 弱信号: "系统里有 What-if 功能"

### B. 痛点和挫折 (3 题)

**Q4: 在产能规划过程中，最让你头疼的事情是什么？**

> 目标: 发现核心痛点
> 强信号: "数据不一致"、"版本混乱"、"找不到谁改了什么"
> 弱信号: "没什么特别头疼的"

**Q5: 你有没有遇到过"事后才发现产能不够"或者"预测和实际差太多"的情况？能举个例子吗？**

> 目标: 了解决策失误的代价
> 强信号: "去年 Q3 我们以为产能够，结果临时外协花了 XXX 万"
> 弱信号: "还好，一般都能预测准"

**Q6: 当你发现数据有问题（比如预测和产能对不上），你通常怎么处理？**

> 目标: 了解数据质量处理方式
> 强信号: "我得一个一个 Excel 检查，经常漏掉"
> 弱信号: "系统会自动提醒"

### C. 数据质量和协作 (3 题)

**Q7: 你的产能规划涉及多少人协作？他们分别负责什么？协作过程中有什么摩擦？**

> 目标: 了解协作需求
> 强信号: "Sales 给预测、Planning 给产能、Finance 给目标，经常对不上"
> 弱信号: "我自己一个人做"

**Q8: 你有没有遇到过"不知道谁改了数据"或者"版本搞混了"的情况？**

> 目标: 了解版本管理痛点
> 强信号: "上个月 Sales 改了预测没告诉我，结果我的分析全错了"
> 弱信号: "我们有版本管理"

**Q9: 你怎么确保你的分析数据是准确和完整的？有没有专门的数据检查流程？**

> 目标: 了解数据质量意识
> 强信号: "我会手动检查一遍，但经常漏"
> 弱信号: "系统会自动检查"

### D. 决策过程 (3 题)

**Q10: 当你需要向管理层汇报产能风险或 BP 达成情况时，你通常准备什么材料？花多长时间？**

> 目标: 了解报告需求
> 强信号: "我每次要做 20 页 PPT，花一整天"
> 弱信号: "系统自动出报告"

**Q11: 管理层最关心什么指标？他们问得最多的问题是什么？**

> 目标: 了解决策关注点
> 强信号: "BP 达成率、产能利用率、客户集中度"
> 弱信号: "各种指标都看"

**Q12: 你有没有用过 AI 工具来辅助产能分析？如果有，效果如何？如果没有，为什么？**

> 目标: 了解 AI 接受度
> 强信号: "我试过 ChatGPT，但数据安全不敢用"
> 弱信号: "我们公司不让用 AI"

### E. 工具和预算 (3 题)

**Q13: 你目前用什么工具做产能规划？对这些工具最满意和最不满意的是什么？**

> 目标: 了解替代方案
> 强信号: "Excel 太慢"、"ERP 的产能模块太重"、"BI 只能看不能算"
> 弱信号: "现在的工具够用了"

**Q14: 如果有一个工具可以让你在 5 分钟内完成 What-if 分析，自动生成报告，并且数据安全，你愿意试试吗？**

> 目标: 测试付费意愿
> 强信号: "当然愿意！什么时候可以试？"
> 弱信号: "我得先看看"

**Q15: 如果这个工具是免费试用的，你愿意导入你的真实数据来测试吗？**

> 目标: 测试真实使用意愿
> 强信号: "可以，我先用一个项目试试"
> 弱信号: "我不敢用真实数据"

## 三、不要问的问题

### 避免引导性问题

| ❌ 不要问 | ✅ 应该问 |
|----------|----------|
| "你觉得 AI 分析有用吗？" | "你有没有用过 AI 工具辅助分析？" |
| "你需要一个新工具吗？" | "你目前用什么工具？有什么不满意的地方？" |
| "你愿意为这个功能付费吗？" | "如果这个工具可以节省你 X 小时，你觉得值多少？" |
| "我们的产品比 Excel 好吗？" | "你用 Excel 做 What-if 分析时遇到什么困难？" |

### 避免假设性问题

| ❌ 不要问 | ✅ 应该问 |
|----------|----------|
| "如果有一个工具可以自动检测数据质量问题，你会用吗？" | "你有没有遇到过数据不一致的情况？怎么处理的？" |
| "如果可以一键生成报告，你会用吗？" | "你准备报告材料通常花多长时间？" |

### 避免技术术语

| ❌ 不要问 | ✅ 应该问 |
|----------|----------|
| "你需要 DQ remediation flow 吗？" | "当你发现数据有问题，你怎么修复？" |
| "你需要 scenario planning 功能吗？" | "你怎么做 What-if 分析？" |

## 四、如何识别强需求信号

### 语言信号

- **主动描述痛点**: "我每个月花 3 天在 Excel 上"
- **表达挫败感**: "每次都要检查好几遍，还是漏掉"
- **询问何时可用**: "这个什么时候可以试？"
- **主动分享案例**: "上个月我就遇到这个问题"
- **表达紧迫感**: "我们真的需要这个"

### 行为信号

- **主动追问细节**: "这个功能怎么实现的？"
- **提出具体需求**: "能不能支持 XXX？"
- **表示愿意试用**: "我可以先用一个项目试试"
- **询问价格**: "这个多少钱？"
- **推荐给同事**: "我同事也需要这个"

### 强信号清单

| 信号 | 含义 |
|------|------|
| "我每个月花 X 小时在这件事上" | 真实痛点 |
| "我上个月就遇到这个问题" | 近期案例 |
| "这个什么时候可以试？" | 紧迫需求 |
| "我可以导入真实数据试试" | 高意愿 |
| "这个功能我马上就能用" | 即时价值 |
| "我同事也需要" | 传播潜力 |

## 五、如何识别伪需求

### 伪信号

| 信号 | 含义 |
|------|------|
| "这个挺好的，但我现在够用" | 没有痛点 |
| "我可以试试，但不着急" | 低优先级 |
| "我得先问问领导" | 不是决策者 |
| "这个功能很酷，但我不确定能不能用" | 没有具体场景 |
| "我觉得这个有用，但我自己不用" | 不是目标用户 |

### 伪需求特征

1. **没有具体案例**: 说不出最近一次遇到问题是什么时候
2. **没有时间投入**: 说不清每个月花多少时间在这上面
3. **没有决策权**: 不是最终使用或购买的人
4. **没有紧迫感**: "以后再说"
5. **没有具体场景**: "听起来不错，但我不确定什么时候用"

## 七、Demo 后追问问题

### 演示后立即问

1. **"刚才演示的功能，哪个对你最有用？为什么？"**
   > 目标: 确认最打动人的功能

2. **"如果这个功能现在就能用，你第一个会用在什么场景？"**
   > 目标: 确认具体使用场景

3. **"你觉得这个工具和你现在用的方式比，最大的区别是什么？"**
   > 目标: 确认价值主张是否清晰

4. **"有什么功能是我没演示但你觉得很重要的？"**
   > 目标: 发现遗漏的需求

5. **"如果要导入你的真实数据，你觉得最大的障碍是什么？"**
   > 目标: 确认导入意愿和障碍

### 深入追问

6. **"你刚才提到的 [具体痛点]，能再详细说说吗？"**
   > 目标: 深入了解痛点细节

7. **"你觉得这个工具应该多少钱合适？"**
   > 目标: 确认价格预期

8. **"如果要向你的领导推荐这个工具，你会怎么说？"**
   > 目标: 了解用户的价值表达方式

## 八、是否愿意导入真实数据试用的判断标准

### 强意愿信号

| 信号 | 含义 |
|------|------|
| "我可以先用一个项目试试" | 有具体试用计划 |
| "我先用 Q3 的数据试试" | 有明确时间范围 |
| "我先导入一个客户的数据" | 有明确范围控制 |
| "我同事可以帮我一起测试" | 有团队支持 |
| "什么时候可以开始？" | 有紧迫感 |

### 弱意愿信号

| 信号 | 含义 |
|------|------|
| "我得先问问领导" | 不是决策者 |
| "我担心数据安全" | 有安全顾虑 |
| "我得先清理一下数据" | 数据没准备好 |
| "我最近太忙了" | 没有时间 |
| "我先看看别人用得怎么样" | 等待观望 |

### 试用意愿判断矩阵

| 决策权 | 数据准备 | 时间 | 试用意愿 |
|--------|---------|------|---------|
| 有 | 有 | 有 | ⭐⭐⭐ 立即试用 |
| 有 | 有 | 无 | ⭐⭐ 安排时间 |
| 有 | 无 | 有 | ⭐⭐ 先准备数据 |
| 无 | 有 | 有 | ⭐ 需要领导批准 |
| 无 | 无 | 无 | ❌ 暂不适合 |
