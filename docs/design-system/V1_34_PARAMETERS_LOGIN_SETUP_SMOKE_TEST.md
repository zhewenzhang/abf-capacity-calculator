# v1.34.0 Parameters / Login / Setup 冒烟测试脚本 (Smoke Test Script)

本脚本用于指导 QA 或开发人员在 v1.34.0 UI 标准化重构后，通过浏览器手动验证相关页面的核心功能与视觉表现，以确保无功能退化和排版崩塌。

---

## 測試場景一：Parameters (參數設定頁) 功能與視覺驗收

### 1. 頁面加載與佈局走查
- **操作步驟**：
  1. 登入系統，導航至 `/parameters`（或點擊側邊欄的“參數設定”/“Parameters”）。
  2. 檢查頁面加載時的 Loading 狀態是否平滑、無閃爍。
- **預期結果**：
  - 頁面加載完畢後，頂部顯示統一的 `PageHeader`，且帶有精美的麵包屑與“參數設定”標題。
  - 頁面主體包含四個獨立卡片：良率矩陣、基板設計參數、幣別與匯率設定、營業目標設定。
  - 頁面整體套用 `abf-page` 容器，四周留白 (padding) 與 Dashboard 保持完全一致。

### 2. 唯讀模式與權限防護驗證
- **操作步驟**：
  1. 使用 **Viewer (唯讀)** 帳號進入該頁面。
- **預期結果**：
  - 頁面頂部顯示“唯讀模式”的 Alert 警告橫幅（使用系統統一的 `abf-alert-page` 樣式）。
  - 頂部 ActionBar 中的“保存參數”及“還原預設值”按鈕皆為置灰 (`disabled`) 狀態，無法點擊。
  - 所有輸入框、表格單元格內的 `InputNumber`、Radio 選項皆為唯讀或置灰狀態，Viewer 絕對無法修改任何參數。

### 3. 編輯模式與參數保存功能驗證
- **操作步驟**：
  1. 使用 **Editor (編輯)** 或 **Admin (管理員)** 帳號進入頁面。
  2. 在“良率矩陣”表格中，任意修改某個尺寸在某個層數的良率值（例如將 `small` / `4-8L` 的 `0.98` 改為 `0.95`）。
  3. 在“基板設計參數”中，修改“工作天數”或“基板長度”等數值。
  4. 點擊頂部 ActionBar 的“保存參數”按鈕。
- **預期結果**：
  - 保存按鈕顯示 Loading 動畫。
  - 右上角彈出“Parameters saved”的成功提示 (AntD message)。
  - 重新刷新頁面，確認剛才修改的數值正確保存並回填，證明保存功能 100% 正常。

### 4. 還原預設值及核心資料保護驗證 (極其重要)
- **操作步驟**：
  1. 在“幣別與匯率設定”或良率中修改數值。
  2. 點擊頂部 ActionBar 的“還原預設值”按鈕。
  3. 在彈出的 Popconfirm 氣泡確認框中點擊“確認”。
- **預期結果**：
  - 良率矩陣與基板參數被成功重置為系統預設值（例如良率均為系統預設、工作天數為預設天數）。
  - **BP 營業目標 (bpTargets) 絕對未被清空或覆蓋**。
  - 為了驗證這一點，請在重置後導航至 `/bp-targets`，確認原有的各年度營業目標資料依舊完好無損，未受任何影響。

### 5. 匯率折算模式切換驗證
- **操作步驟**：
  1. 在“幣別與匯率設定”中，切換“匯率折算模式”為“常數匯率”。
  2. 檢查是否顯示 USD to TWD 和 USD to CNY 常數輸入框。
  3. 切換“匯率折算模式”為“年度匯率”。
  4. 檢查常數輸入框隱藏，並流暢顯示“年度 USD to TWD 匯率”和“年度 USD to CNY 匯率”的兩張小表格。
  5. 任意修改某年度的匯率，點擊“保存參數”並刷新驗證。
- **預期結果**：
  - 模式切換邏輯無縫，無排版錯位。
  - 表格寬度適中，數字输入框對齊，保存與讀取完全正常。

## 測試場景二：LoginPage (登入頁) 視覺驗收

### 1. 登入介面視覺與排版
- **操作步驟**：
  1. 登出系統（或在無痕瀏覽器中打開應用），進入 `/login`。
- **預期結果**：
  - 登入卡片在頁面垂直及水平居中。
  - 背景色調高雅，不再是死板的灰白色，而是符合設計系統全局的中性偏灰背景。
  - “Google 登入”按鈕樣式亮眼，寬度撐滿 (block) 且帶有 Google 彩色或設計系統規範圖標。

### 2. 登入流暢性與異常攔截
- **操作步驟**：
  1. 點擊“Google 登入”按鈕。
  2. 快速點擊按鈕，觀察防重覆點擊機制 (Loading 狀態)。
- **預期結果**：
  - 按鈕點擊後立即呈現 `loading` 狀態，且無法重覆點擊，有效防止二次請求。
  - 成功登入後，系統應順暢重定向至 Dashboard 頁。
  - （可選驗證）人為中斷登入流程，確認錯誤 Alert 提示以精美的樣式出現在按鈕上方，並可正常關閉或在下一次點擊時消失。

## 測試場景三：SetupPage (初始化配置頁) 視覺驗收

### 1. 配置提示頁視覺
- **操作步驟**：
  1. 修改本地環境變量使其缺失，或手動導航至 `/setup`（若路由支援）。
  2. 檢查配置提示頁的視覺表現。
- **預期結果**：
  - 頁面正中顯示 Result 大圖標及“初始化配置”標題。
  - 步驟說明文字（Step 1 - Step 6）字型與段落排版整潔。
  - 環境變量配置程式碼塊 (`.env` template) 邊角圓潤，字體使用 Monospace 等寬字體，背景顏色高雅且支援橫向滾動，無任何溢出或包裝破損。
