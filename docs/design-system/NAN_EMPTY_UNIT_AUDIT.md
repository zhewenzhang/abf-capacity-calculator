# NaN、空值与单位漏洞白盒审计报告 (NaN/Empty/Unit Audit)

为了维护财务与产能核算的绝对精度，防范隐秘的数学分母除零崩溃、NaN 空值泄露以及物理单位混淆，本报告针对 ABF 产能计算器中所有涉及分析、对比、快照的表格及卡片执行深度专项审计。

---

## 🚨 4 大高风险计算溢出场景大剖析

在涉及年度目标达成率（Attainment %）、设备利用率（Utilization %）以及良率的后台运算引擎中，开发团队 (CC) 极易在以下四个极限场景下触发 `NaN` (Not a Number) 或 `Infinity` (正无穷) 逻辑溢出：

### 1. 年度 BP Target 缺失下的营收达成率计算
- **触发路径**：用户在 [BP Targets] 独立页中未填报 2027 年目标（数据层为 `null`），随后直接切换到 `BP Analysis` 分析报表。
- **溢出机理**：计算公式 `Actual Revenue / BP Target`。如果代码仅用强转 `Number(null) -> 0`，会导致除数为 0。在 JS 中，`Revenue / 0` 会爆出 `Infinity`（正无穷大），导致前端呈现 `Infinity%` 或者是 `NaN%`，引发界面白屏或图表数据畸变。

### 2. 瓶颈设备额定产能（Rated Capacity）设为 0 下的产能超载运算
- **触发路径**：用户在 `Capacity Lab` 中将某台不常用制程设备的额定产能故意置为 `0`（表示该设备本月不开机，产能为零）。
- **溢出机理**：`Required Hours / Rated Capacity`。由于分母为 0，这会导致该制程的产能利用率瞬时飙升为 `Infinity%`，在生成利用率雷达图和柱状图时触发 Canvas 溢出异常，引发系统运行白屏。

### 3. 良率（Yield %）配置未录入下的 SKU UPP 联动计算
- **触发路径**：用户在产品列表中新增一个 SKU 规格，但未在此 SKU 下配置其良率分类。
- **溢出机理**：良率估算在缺失参数时返回 `null`。当下游进行 `UPP * Yield %` 运算以评估折合良率产能时，如果直接用 `upp * null`，在 JS 中将产生 `NaN` 并顺着数据流向上逐层污染，最终导致营收报表汇总行沦为一连串刺眼的 `NaN Million TWD`。

### 4. 表格中 `Million TWD` 与 `TWD` 元单位混淆风险
- **触发场景**：Dashboard 顶部的累计目标达成展示中直接标示 `總營收目標: 12,500`，未标明单位。
- **灾难风险**：业务人员极易将其误判为 12,500 元台币或是 12,500 美元。如果在计算 SKU 售价（通常为 USD 美元，如 `3.5 USD`）乘以销售量时，未对汇率乘数 `usdToTwd` 进行百万级换算（$\times 10^{-6}$），直接与 Million TWD 对比，会导致对比数据相差 1,000,000 倍，瞬间造成虚假达成率大事故。

## 🎨 建议的全局数据呈现格式标准 (The Standard Formatting)

为了让系统完全跨过“财务级成熟”分界线，必须对系统所有数据过滤格式化逻辑强制统一执行以下标准：

| 原始数据值 (Database Value) | 展示格式要求 (UI Format Output) | 技术细节与安全防线规约 |
| :--- | :--- | :--- |
| **未录入的 null / undefined** | `—` (灰色淡雅短横线) | 严禁显示为空白字符串 `""`，防范与 0 语义发生混淆。 |
| **发生除零溢出的 Infinity / NaN** | `— (無目標)` 或 `0%` | 公式端捕获：`if (!target \|\| target === 0) return null;`，回弹优雅显示。 |
| **百分比利用率 / 达成率** | 整数百分比，如 `89%` | 使用 `Math.round(val * 100)` 统一取整，严禁裸露 `88.888888%` 冗余浮点数。 |
| **台币营业总额** | 带千分位，如 `12,500` | 使用 `val.toLocaleString('zh-TW')` 格式化，表头统一显示 `(百萬新台幣)`。 |
| **美金 SKU 单价** | `$3.50` | `val.toFixed(2)` 强锁小数点后两位，前缀显式美金符号。 |

## 🧪 4 组极限值白盒测试用例 (QA Test Cases)

请 QA 团队与开发团队 (CC) 建立以下极限测试用例集，以保障数据计算的极客级健壮性：

### TestCase-01：【未考核年份空值熔断测试】
- **输入数据**：
  - `ProjectParameters.bpTargets.yearlyRevenueTargetsMillionTwd = { "2026": 12000, "2027": null }` 
- **操作步骤**：切换年份选择器到 `2027` 年，进入 Results 分析页。
- **预期结果**：2027 年营收目标单元格优雅呈现 `—`。达成率计算模块自动返回 `—`，柱状图与雷达图中自动剔除 2027 年，不参与求和与分母运算，0 崩溃。

### TestCase-02：【设备产能置零超载防爆测试】
- **输入数据**：
  - 瓶颈设备 Rated Capacity = `0`。
  - Required Hours = `350` (小时)。
- **操作步骤**：进入 Capacity 利用率分析表格。
- **预期结果**：产能利用率单元格捕获分母 0 溢出，优雅显示为 `— (停機)`，标以灰色 Tag 提示，雷达图不崩溃，0 溢出。

### TestCase-03：【未配置良率下 derived 衍生字段稳定性测试】
- **输入数据**：
  - 新增 SKU_Omega，良率分类 = `null`，芯片尺寸 Length = `25`，Width = `25`。
- **操作步骤**：进入 Products 列表页。
- **预期结果**：derived 出来的 UPP 正常计算显示为 `85`，良率预估 Yield 单元格显示为 `—`，芯片折合有效产能计算函数返回 `null`，整表数据无 NaN 泄漏。

### TestCase-04：【多币种美元中转累计对账精度校验】
- **输入数据**：
  - 销售预测量 = `100,000 (件)`，单价 = `3.5 (USD)`。
  - 固定折算参数 `usdToTwd` = `32.5`。
  - BP 营业目标 = `11.375 (百万台币)`。
- **运算过程验证**：
  - 标准化折算第一步：累计营收 = $100000 \times 3.5 = 350,000 \text{ USD}$。
  - 折算台币第二步：$350000 \times 32.5 = 11,375,000 \text{ TWD}$。
  - 折算百万台币第三步：$11375000 \times 10^{-6} = 11.375 \text{ Million TWD}$。
- **预期结果**：Results 表格中算出的 2026 年实际营收与 BP 目标（11.375 百万）实现 **100.00% 零分差完美对账**，无由于时点或计算精度丢失导致的 0.01% 对账残差。
