# ABF 设计系统规范第 1 版 (ABF UI Design Rules v1.0)

本规范是 ABF 产能计算器前端开发与界面设计的**硬性红线标准**。后续无论由协同团队 (CC)、旁路团队 (AGY) 还是自动化 AI 进行代码编写，均必须严格死守本规范定义的视觉与交互边界。

---

## 📐 1. Layout Rules (布局规范)

### A. 页面四大基本布局基线
- **Dashboard (只读大屏页)**：采用精美 `Grid` 宫格网格布局，第一排固定为 Metric 汇总指标卡片，下方为产能负荷图表与快速跳转卡。
- **Results (数据分析页)**：采用 `Tabs` (标签分类) + `AntD Table` 结构，页头固定带有面包屑及数据置信度警示卡（DataCaveatAlert）。
- **Input pages (电子表格录入页)**：采用 `Full-Width` (全宽网格流)，以 react-datasheet-grid 二维大表填满视口，页头摆放 Year Selector，页面下方或顶部固定浮现 `DirtyStateBar` 脏提示条。
- **Settings pages (低频偏好设置页)**：采用 `Row-based Cards` (单列配置卡片流)，表单元素合理分组在精致的卡片内。

### B. 卡片 (Card) 使用边界与嵌套红线
- Card 只能作为一级的容器组件，用来对逻辑相关的字段或图表进行物理分组。
- > [!CAUTION]
  > **红线：严禁在 Card 内部再嵌套 Card (No Nested Cards)！**
  > 卡片内套卡片会导致多重投影（box-shadow）和内边距（padding）层层堆叠，产生严重的视觉斑驳，使用户的视觉焦点发生混乱。

### C. Section 间距规范
- 页面一级 Section（板块）垂直间距固定使用 `24px`。
- 模块内部各元素（如表头与网格、卡片内各输入框）垂直间距固定使用 `16px`。

## 📊 2. Table Rules (表格规范)

### A. 表格引擎场景严密划分
- **Ant Design Table 适用**：Results 分析表、Dashboard 历史列表、只读参数列表。
- **react-datasheet-grid 适用**：BP Targets、Capacity Lab、Products Lab 等中轻量 Excel-like 横向高频录入。
- **TanStack Table + Virtual (未来) 适用**：当单空间 SKU 数突破 **500行** 或总单元格突破 **12,000个** 时的海量销售大表录入。

### B. 对齐、数值、空值与滚动规则
- **文字对齐**：SKU Code、设备名称等文本类列一律 **靠左对齐 (text-align: left)**；月度销售额、利用率百分比等数值列一律 **靠右对齐 (text-align: right)**，并使用等宽数字（font-variant-numeric: tabular-nums）以便上下快速 scan 对账。
- **空值重绘**：数据为 `null` 或未考核年份时，单元格一律渲染为优雅的灰色短横线 `—`，严禁直接丢空 `""`，更不允许出现 NaN 字符。
- **单位露出**：表格表头必须强制带括号标示物理单位，如 `額定產能 (萬片)`、`月預測量 (件)`。
- **横向滚动保护**：凡列数超过 8 列的宽表，必须用外层容器包裹并设定 `scroll={{ x: 'max-content' }}`，严禁窄屏幕下表头被迫折行。

## 💾 3. Input Rules (录入与脏状态控制)

### A. Save / Discard 与脏状态控制线
- 二维录入网格修改后必须触发全局 `DirtyStateBar`（未保存提示条）。
- 提示条中 [保存 (Save)] 按钮在正常下为置灰态，仅在有未存脏数据且全表校验无负数/非法值时高亮激活；[放弃 (Discard)] 点击后在 `0ms` 延迟内进行内存快照物理回滚。
- 用户尝试在此脏状态下切换页面路由，必须强行触发项目统一封装的 `RouterGuard` 阻止跳转并弹窗提示。

### B. 【硬性红线】禁止 Autosave 场景
- > [!IMPORTANT]
  > **严禁在 Forecasts 销售预测大表及良率良率配置表中使用任何失焦自动保存 (Auto-save on blur)！**
  > 高频打字录入会引发成千上万次 Firestore set 发包，直接导致企业月度资费爆炸与严重的并发写入覆盖。

## 🎨 4. Status / Tag Rules (标签与语义色规范)

### A. 四大语义色彩标准线
- 🟢 **Success (成功/达标)**：`#52c41a` (AntD Green)。用于 BP 达成率 $\ge 100\%$，产能负载安全。
- 🟡 **Warning (警示/饱负荷)**：`#faad14` (AntD Orange)。用于产能利用率处于 $[85\%, 100\%]$ 饱负荷状态。
- 🔴 **Error (超载/缺失)**：`#ff4d4f` (AntD Red)。用于产能利用率 $> 100\%$ 超载，或 BP 严重未达标。
- 🔵 **Info (快照/未激活)**：`#1677ff` (AntD Blue)。用于 Snapshot 状态标示或说明性 Tag。

---

## 💵 5. Currency / Unit Rules (币种与精度规范)

- **显示精度**：
  - 美元（USD）数值一律保留至小数点后 **2 位**。
  - 新台币（TWD）数值一律四舍五入保留为 **整数**（无小数）。
  - 利用率一律保留为整数百分比，如 `89%`。
- **年度 BP Target 规范**：其在数据库与 UI 一律采用新台币计价，且固定以 **百万新台币 (Million TWD)** 为唯一标准物理单位。

---

## 🌐 6. i18n Rules (国际化与繁体中文规范)

- **0 硬编码红线**：UI 代码中绝对不应裸露中英文原始字符串，必须全部以 `t('key')` 从语言包获取。
- **Parity 对称原则**：在 `en.ts` 中每新增一个翻译 Key，必须在 `zhTW.ts` 同一树层级 1:1 追加对应繁体中文翻译，防国际化缺失。
- **地道繁体中文用词标准**：
  - 销售预测 -> **銷售預測** (避开“銷預”)。
  - 良率 -> **良率** (避开“成品率”)。
  - 产能 -> **產能** (避开“生產能力”)。
  - 快照对比 -> **快照比對** (避开“對比”)。

---

## 🤝 7. Analysis Trust Rules (数据可信度规范)

- **数据源透底标示**：在 Results 页面中，只要展示了预测对比，必须在页头显示精致的信息条，明示：“*本报表基于 deterministic (确定性逻辑计算)，非 AI 幻觉生成，数据真实可追踪。*”
- **免责声明 (Data Caveat)**：AI risk 简报头部固定渲染免责水印。
- **数学比例分摊保护**：在显示 Top Changes 时，UI 文字与图表 Title 必须写为 `缺口分攤貢獻度 (Attribution)`，**绝对禁止**写成 `因...導致未達標 (Causality)`，防止因果归因谬误误导高管。
