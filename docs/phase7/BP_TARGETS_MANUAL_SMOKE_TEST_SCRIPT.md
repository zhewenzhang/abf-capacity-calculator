# BP Targets 独立页面手动冒烟测试脚本 (Manual Smoke Test Script)

本脚本用于在 **v1.25.0 BP Targets 独立页面** 开发完成后进行全链路的功能质量与权限安全验证。执行测试时请严格按照前置条件与步骤进行核对。

---

## 🧪 测试用例矩阵

### TC-01：Personal Mode (个人模式) 下 BP 目标全生命周期编辑测试
* **前置条件**：
  - 用户登录系统，且当前所处空间为 **Personal Mode (个人空间)**。
  - 确保网络连接正常。
* **测试步骤**：
  1. 点击导航栏的 [BP 营业目标] 菜单，进入独立编辑页面。
  2. 观察当前表格，在任意年份单元格内（如 `2026` 年）双击，修改目标值为 `1800`。
  3. 观察页面顶部或底部是否立即浮现带有 “未保存修改” (Dirty State) 的提示条，以及 [保存 (Save)] 和 [放弃 (Discard)] 按钮是否亮起。
  4. 点击 [保存 (Save)] 按钮。
  5. 观察提示条是否消失，[保存] 按钮是否恢复置灰状态。
  6. 重新刷新整个浏览器网页。
* **预期结果**：
  - 步骤 3 中：Dirty 状态提示灵敏出现，按钮变为可用。
  - 步骤 5 中：保存成功提示弹出，状态条归位，按钮置灰。
  - 步骤 6 中：页面刷新后，`2026` 年的目标值依然牢牢锁死在 `1800`，说明写入 Firestore 数据库持久化成功。

### TC-02：Workspace Owner 角色保存权限验证
* **前置条件**：
  - 系统中有两个账号（User A 与 User B）。
  - 创建一个共享工作空间 `WS-Alpha`。
  - **User A** 为该工作空间的 **Workspace Owner (所有者)**。
* **测试步骤**：
  1. User A 登录并切换到 `WS-Alpha` 工作空间。
  2. 点击进入 [BP 营业目标] 页面。
  3. 在 `2027` 年对应的单元格中填入新数值 `2500`。
  4. 点击 [保存 (Save)]。
* **预期结果**：
  - 允许进行双击编辑。
  - [保存] 按钮高亮且点击后可以成功提交，界面提示“保存成功”。
  - 数据库字段被成功写入。

### TC-04：Workspace Viewer 角色安全隔离与只读校验
* **前置条件**：
  - 共享工作空间 `WS-Alpha` 中。
  - **User C** 被邀请，且仅被赋予 **Viewer (只读观察员)** 权限。
* **测试步骤**：
  1. User C 登录并切换到 `WS-Alpha` 工作空间。
  2. 点击进入 [BP 营业目标] 页面。
  3. 观察页面中是否能够找到 [保存 (Save)] 和 [放弃 (Discard)] 按钮。
  4. 双击 `2026` 或任意年份单元格，尝试输入新的数值。
  5. 尝试使用键盘快捷键（如 `Ctrl + V` 或 `Backspace`）修改数据。
* **预期结果**：
  - 页面上绝对不应显现 [保存] 与 [放弃] 按钮（或者按钮呈现彻底禁用态，并有 Tooltip 标明：“只读空间无编辑权限”）。
  - 双击任何单元格均无法进入编辑态，不闪烁输入光标，单元格背景维持只读色泽。
  - 任何键盘擦除、粘贴指令均被彻底拦截，数据毫发无伤。

### TC-05：Excel 跨页横向多行粘贴兼容性测试
* **前置条件**：
  - 电脑上打开 Excel、WPS 或 Google Sheets 软件。
  - 在 Excel 单元格中横向输入一行数据：`1500  1800  2000  2400`（按 Tab 键隔开）。
  - 复制该行数据。
* **测试步骤**：
  1. 打开 ABF 计算器的 [BP 营业目标] 页（Editor 及以上权限）。
  2. 单击选中表格的首个输入单元格（例如 `2024` 年）。
  3. 按下快捷键 `Ctrl + V`（Mac 系统使用 `Cmd + V`）。
  4. 观察数据在表格中的填充表现。
* **预期结果**：
  - `1500` 填入 `2024`，`1800` 填入 `2025`，`2000` 填入 `2026`，`2400` 填入 `2027`，实现完美的横向单元格自动铺开。
  - 严禁出现把整个 `1500 1800 2000 2400` 字符串强制挤入 `2024` 年单元格的情况。

### TC-06：非数字输入与负数边界拦截测试
* **前置条件**：
  - 处于具有编辑权限的工作空间中。
* **测试步骤**：
  1. 在 `2026` 年单元格双击输入 `-500`，失焦。
  2. 在 `2027` 年单元格输入字母 `ABC`，失焦。
  3. 观察单元格的视觉反馈。
  4. 观察 [保存 (Save)] 按钮状态。
  5. 尝试强行点击 [保存 (Save)]（若按钮亮起）。
* **预期结果**：
  - 输入 `-500` 后，单元格周边应标红，或立即回弹报错，[保存] 按钮彻底置灰禁用。
  - 输入 `ABC` 时，单元格应直接静默过滤字母（无法输入），或在失焦时弹出警告并回滚至输入前的值。
  - 绝不应允许将 `-500` 或 `ABC` 保存写入数据库。

### TC-07：下游报表（Dashboard / Results）同步联动测试
* **前置条件**：
  - 处于可编辑空间，且已记录当前的 `2026` 年 BP 营业目标为 `1000`（万）。
  - 观察 Results 页面下 `BP Analysis` 中显示的 BP 达成率数值。
* **测试步骤**：
  1. 切换至 [BP 营业目标] 页面，将 `2026` 年目标修改为 `2000`（直接翻倍），点击 [保存 (Save)]。
  2. 不刷新浏览器，直接通过左侧导航切换到 [Results] -> [BP Analysis] 报表页面。
  3. 观察该报表中 2026 年的 BP Target 设定值以及 BP 达成率百分比（Attainment %）。
* **预期结果**：
  - 步骤 3 中，`BP Analysis` 报表中的 target 值已瞬间自动刷新为 `2000`。
  - BP 达成率百分比已随之重新计算并同步降低，无任何渲染延迟或需要手动 F5 刷新的滞后。

### TC-08：Discard (放弃更改) 瞬时无损回滚测试
* **前置条件**：
  - 处于可编辑空间，原始 `2026` 年目标为 `1200`。
* **测试步骤**：
  1. 进入 [BP 营业目标] 页面。
  2. 将目标值狂刷修改为 `99999`。
  3. 观察到 Dirty 状态提示条亮起。
  4. 绝不点击保存，直接点击 [放弃修改 (Discard)]。
* **预期结果**：
  - 输入框内的 `99999` 瞬间被擦除，重置回 `1200`。
  - Dirty 状态提示条自然消失，[保存] 与 [放弃] 按钮恢复置灰禁用。

### TC-09：多语言 (i18n) 语言包无缝切白测试
* **前置条件**：
  - 位于应用主界面。
* **测试步骤**：
  1. 进入 [BP 营业目标] 独立页。
  2. 点击右上角语言切换，将系统语言切至 **English**。
  3. 细致扫描：页面标题、表格表头（若有）、提示文字（如“未保存更改”等）、保存/丢弃按钮、报错 Tooltip。
  4. 再次点击，将系统语言切至 **繁體中文 (zh-TW)**。
  5. 重新扫描相同文字区域。
* **预期结果**：
  - 切至 English 时，100% 呈现流利的英文。
  - 切至繁体中文时，100% 呈现规范的繁体中文字样（如“儲存”、“捨棄”、“尚未儲存的變更”等）。
  - 严厉杜绝出现如 `changeReview.recommendedReason` 类似的翻译键直接裸露在 UI 上的故障（i18n Key Leak）。
