# BP Targets 独立页面验收清单 (Acceptance Checklist)

本清单用于指导并约束 **v1.25.0 BP Targets 独立页面 + 表格标准化** 功能的开发与验收。研发团队 (CC) 在提请合并前必须逐项核对并确保 100% 达标。

---

## 📌 核心验收要点

### 1. 路由与菜单导航验收
- [ ] **目标**：BP Targets 必须作为独立的一级或二级功能页面呈现。
- [ ] **规范**：
  - 左侧侧边栏或顶部导航中，应有独立的“BP 营业目标”或 “BP Targets” 导航入口。
  - 路由地址建议为 `/parameters/bp-targets` 或 `/bp-targets`。
  - 页面标题、面包屑导航需与菜单名称保持一致，在中英文切换时翻译正确。

### 2. BP Targets 数据存储位置与格式验收
- [ ] **目标**：数据层保持纯净，不得私自更改 Firestore schema 或新建 Collection。
- [ ] **规范**：
  - 所有在该页面录入的年度营业目标数据，其在数据库中的存储路径必须严格维持在：
    `ProjectParameters.bpTargets.yearlyRevenueTargetsMillionTwd`
  - 格式必须符合现有的 Map 结构，键（key）为年份字符串（如 `"2026"`），值（value）为数字类型（如 `12500`），单位为 **百万新台币 (Million TWD)**。
  - 严禁为了该页面的临时需要将数字格式改为带单位的字符串或私自嵌套其他冗余对象。

### 3. 表格年份横向排列渲染验收
- [ ] **目标**：符合财务与高管使用 Excel 的直观体验。
- [ ] **规范**：
  - 表格的表头应为年份（如 `2024`, `2025`, `2026`, `2027` ... 顺序递增），行数据为营业目标值。
  - 年份从左至右横向一字排开，而非纵向行排开。
  - react-datasheet-grid 必须正确渲染出这一横向表格，且年份列宽比例适中，无溢出或滚动条异常。

### 4. react-datasheet-grid 组件标准交互验收
- [ ] **目标**：表格录入交互符合高水准 Spreadsheet 规范。
- [ ] **规范**：
  - 允许通过键盘快捷键进行单元格间移动：支持 `Tab`（向右移）、`Shift + Tab`（向左移）、`Enter`（向下移或双击进入编辑）以及上下左右方向键。
  - 单元格在聚焦状态下，应有清晰的边框高亮（Ant Design 风格蓝色高亮）。
  - 双击单元格或按回车应能直接进入编辑态，失去焦点或按 `Esc` 退出编辑并保留/丢弃更改。

### 5. Excel 剪贴板横向复制粘贴验收
- [ ] **目标**：支持财务大批量录入。
- [ ] **规范**：
  - 从物理 Excel 或 CSV 软件中横向复制一排数值（如 `1200\t1300\t1400`），在页面表格首个目标单元格粘贴时，必须能够自动将值正确拆分并映射到相邻的年份列中。
  - 严禁将复制的所有年份值全部挤在单个单元格内作为单个字符串保存。
  - 粘贴包含非数字的内容时，应当有友好的截断或格式化过滤机制。

### 6. 输入边界值过滤与非数字拦截验收
- [ ] **目标**：保障底层计算引擎数据安全性，防空值或非法数据崩溃。
- [ ] **规范**：
  - **空值处理**：允许用户清空某年份目标。清空后，在保存时数据层对应年份应存为 `null`（或者直接从 Map 中移除该 Key），而不是保留为字符串 `""` 或存为 `0`。
  - **负数拦截**：营业目标不可为负数。若用户输入负数，在单元格失焦时应标红并触发 Validation 报错，Save 按钮必须置灰禁用。
  - **非数字过滤**：若用户尝试手动输入英文、符号或特殊字符，表格单元格应自动过滤拦截非数字输入，或者失焦时直接重置回原数值并发出提示。

### 7. Dirty State (脏状态) 与保存/丢弃生命周期验收
- [ ] **目标**：提供平滑、安全的防呆机制。
- [ ] **规范**：
  - **脏状态提示**：一旦用户对表格进行过任何实质修改，页面上方或底部应立即浮现“未保存更改”的提示状态（Dirty State），且 [保存 (Save)] 与 [放弃更改 (Discard)] 按钮变为可用状态。
  - **放弃更改**：点击 [放弃 (Discard)]，页面必须立即无损回滚至数据库中最后一次保存的快照数据，且脏状态消失。
  - **保存生效**：点击 [保存 (Save)]，触发 Firebase 写入，成功后脏状态消失，按钮变回置灰态。
  - **防强退**：若页面处于脏状态，用户尝试切换路由或关闭标签页，应弹出标准浏览器或 React Router 拦截弹窗提醒：“您有未保存的更改，确定要离开吗？”。

### 8. 四级角色权限行为矩阵验收
- [ ] **目标**：严守多租户空间安全红线。
- [ ] **规范**：
  - 🔐 **Personal Mode**：作为个人沙盒，用户拥有完全的读写权限，能够自由编辑、保存与放弃。
  - 👑 **Workspace Owner**（空间所有者）：拥有完全的读写权限，Save / Discard 按钮可用，可向 Firestore 提交空间级别的 BP 设定。
  - ✍️ **Workspace Editor**（空间协同编辑）：拥有完全的读写权限，可编辑、可正常保存更改。
  - 👁️ **Workspace Viewer**（只读观察员）：**完全禁写**！react-datasheet-grid 必须设置为只读态（disabled/read-only），不允许双击编辑，[保存] 和 [放弃] 按钮彻底隐藏或禁用。如果用户强行尝试通过终端或 API 提交，后端安全规则（Firestore Rules，尽管本轮不改，但前端必须防守）予以拒绝。

### 9. Parameters 旧页面跳转与平滑退役验收
- [ ] **目标**：用户路径专一，不留混乱入口。
- [ ] **规范**：
  - 原 Parameters 页面中编辑 `ProjectParameters.bpTargets` 的表单或入口必须彻底移除。
  - 替代方案：在该位置摆放一张精致的说明卡片（Alert / Info Card），明确指出：“BP 营业目标已迁移至独立页面管理”，并提供一键跳转至新独立页面的高亮按钮（Button/Link）。

### 10. 全局数据关联与 i18n 翻译对齐验收
- [ ] **目标**：功能连贯，且在中英文下无破损。
- [ ] **规范**：
  - **下游联动**：修改并成功保存 BP Targets 后，无需刷新页面，直接切到 Dashboard、Results、BP Analysis 等分析页面，确认其计算 Gap 与 Target 达成率时能瞬时读取到最新保存的目标值，0 数据漂移。
  - **i18n 1:1 对齐**：BP 独立页面的所有标签（如“年度营业目标”、“单位：百万新台币”、“未保存修改”、“保存”、“丢弃”、“无权限”等）必须在 `en.ts` 和 `zhTW.ts` 中完全对齐，严禁出现任何硬编码英文或未翻译键值。
