# Phase 7 Excel-like 交互与输入 UX 规格说明书

本说明书定义了 **ABF Capacity Calculator** 在 **Phase 7 高频类 Excel 输入工作流** 中的 **UX 交互规格与技术标准 (UX Spec)**。旨在为前端开发团队提供无懈可击、符合工业级决策直觉的表格录入规范，确保其与系统全局的 Ant Design 6 视觉风格保持绝对的协调与专业品位。

---

## 一、 使用者故事 (User Stories)

- **Sales (业务人员)**：
  - “当我要录入一个新的 SKU 时，我希望能够直接点击表格的首行，像用 Excel 一样一口气用 `Tab` 键向右移动，顺畅录入参数，而不需要用鼠标频繁点选输入框。”
  - “当我要为某个 SKU 录入全年的 Forecast 销售数据时，我只要录入 1 月份的片数，然后点击该单元格右下角的‘填充柄’，向右拖拽，即可瞬间把数值平刷复制到 2 月至 12 月（Fill Right），免去我繁琐的逐月敲击。”
- **Capacity Planner (产能规划经理)**：
  - “当机台设备良率调整时，我需要将外部计算好的 200 条设备线、12个月度的产能大表（共计 2400 个单元格）在 Excel 中一次性复制，然后直接在系统的排产表格首格按下 `Ctrl + V`。系统必须在毫秒级自动解析并粘贴，且所有格式、超出范围的脏数据都要有明确的标色和校验提示。”

---

## 二、 Products / Forecasts / Capacity 核心页面输入 UX 规格

### 1. Products 页面：SKU 参数高频录入
- **布局设计**：采用垂直表格列布局。首列固定，显示 SKU 编码，便于纵向核算。
- **键盘导航 (Keyboard Navigation)**：
  - **`Arrow Up / Down / Left / Right` (方向键)**：直接在单元格选区（Cell Selection）间顺畅位移。
  - **`Enter` 键**：进入当前单元格的编辑模式；再次按下 `Enter` 键确认编辑并自动向下移动一格。
  - **`Tab` 键**：确认当前单元格编辑并向右移动一格；移至行尾时自动换行到下一行的首格。
  - **`Esc` 键**：放弃当前单元格的修改并退出编辑状态。

### 2. Forecasts 页面：横向月份多维排表录入 (灵魂交互)
- **布局设计**：
  - **首部三列固定**：SKU Code | 客户 (Customer) | 设备 (Device) — 这三列在横向滚动时保持固定，保证参考基准不丢失。
  - **横向 12/24 列平铺**：按月（如 `2026-01`, `2026-02` ...）纵向平铺。
- **平刷填充柄 (Fill Right / Fill Down)**：
  - **拖拽行为**：单元格处于被选中（Selected）状态时，其右下角渲染出一个 4x4 像素的微小实心蓝色矩形（填充柄 / Fill Handle）。
  - **横向平刷 (Fill Right)**：鼠标按住填充柄向右拉动，在拖拽过程中，覆盖的月份单元格呈现一层淡蓝色、带虚线边框的临时遮罩；鼠标松开后，**源单元格的 Forecast 数值一键复制平刷到所有覆盖的月份中**。
  - **纵向平刷 (Fill Down)**：按住向下拖拽，将源单元格的单价或数量向下一键平刷给其他 SKU。
- **年份快速复制 (Yearly Replication)**：
  - 每行 SKU 尾部或行首提供快捷「Replicate / 复制」图标。点击后，可弹窗选择“将 2026 全年 Forecast 复制到 2027 对应月份”，实现跨年份一键翻版复制。

### 3. Capacity Plan 页面：大型设备良率与稼动率表
- **布局设计**：机台类别与设备名称固定在左侧，横向展开 12 个月良率和 12 个月稼动率上限。
- **特大型 Clipboard 复制粘贴 (Paste from Excel)**：
  - 当用户在外部 Excel 选中一个矩形区域并复制时，剪贴板内的格式为以 `\t` (Tab 键分隔字段) 与 `\n` (换行符分隔行) 的纯文本块。
  - 用户在系统的良率表格中选中起始单元格，按下 `Ctrl + V`：
    - **自动映射定位**：系统拦截 `paste` 事件，在毫秒级将切割出的文本矩阵，顺着起始单元格按行列顺序填充。
    - **只读写拦截**：如果目标单元格属于只读计算单元（如派生的 Core UPP 或 Yield 估算值），系统**自动忽略对其覆盖写入**，仅在可修改的数据列（如 chipLength, layerCount）进行值注入，保障物理计算不破损。

## 三、 数据校验错误指示与脏状态管理 (Data Validation & Dirty State)

为了防止脏数据污染云端 Firestore，输入系统必须在本地建立起**三级校验和两级脏状态管理**：

### 1. Validation 错误高美感展示 (Validation Errors)
- 当用户录入或粘贴了非法数据（如单价为负数、层数为 0、或者汇率币别不是 USD/TWD/CNY）时：
  - **防崩溃闪烁**：前端**绝对不允许弹窗打断**，而是顺畅接受数据输入。
  - **单元格变色与 Tooltip 提示**：该单元格背景色瞬间渲染为带象征警示的淡橙红色（如 `#fff2f0`），并附带红色虚线边框；鼠标悬浮在变色单元格上时，优雅弹出 Tooltip 明确指出违规的具体校验错误（如：`层数必须在 1 至 30 之间`）。
  - **物理保存禁用**：只要表格中存在任意一处变红的校验错误，页面顶部的「保存 / Save」按钮将物理置为 `disabled`，防止将脏数据强推至 Firestore。

### 2. 脏状态 (Dirty State) 视觉闭环 (Save / Discard)
- **毫秒级本地缓存响应**：表格内高频的修改、单元格平刷、以及 `Ctrl + V` 粘贴，**全部在 React 内存组件的 local state 中发生。严禁在录入单元格失焦时频繁向 Firebase 发送 update/set 请求**！这不仅可以实现绝对流畅的录入，且避免了巨大的 Firestore 读写读写放大。
- **脏状态徽章 (Dirty Badge)**：
  - 只要表格数据与云端拉取的底档不一致，页面顶部显著呈现代表待提交状态的橙黄色**「未保存修改 / Unsaved Changes」**徽章。
  - 提供两个显著按钮：
    - **`[保存 / Save]` (蓝色)**：触发 `batchSaveSKUs` 或对应的 batchSave 逻辑，一键将数据脱敏及规整化后强力持久化到 Firestore，成功后脏标志清空。
    - **`[放弃 / Discard]` (灰色)**：点击后带确认 Popconfirm 提示，确认后一键回滚数据至上次已保存快照，恢复清爽。
