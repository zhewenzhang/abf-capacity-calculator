# v1.36.0 Data Quality Quick Fix Smoke Test Script (冒烟测试剧本)

本剧本模拟一位 ABF 产能规划与决策分析专家，在脏数据多发的真实工作场景下，使用 `v1.36.0` 带来的**数据自愈工作流 (Data Quality Remediation Workflow)**，进行全链路无死角的冒烟测试。

---

## 角色配置与环境准备

* **协同工作区 ID**：`smoke-test-workspace-v136`
* **参与角色**：
  * **角色甲 (Planner-Owner)**: 拥有 `Owner` 权限，负责进行各项数据自愈操作。
  * **角色乙 (Auditor-Viewer)**: 拥有 `Viewer` 权限，负责核实只读隔离和完整知情权。
* **初始脏数据准备**（已通过脚本批量录入）：
  1. SKU 主表录入了一款缺失层数（`layerCount` 缺失）的产品 `SKU-DIRTY-01`。
  2. Forecasts 预测中，包含一条绑定了不存在的 SKU ID (`SKU-GHOST-99`) 的孤儿预测需求。
  3. Forecasts 预测中，2026-03 的某条预测记录，其价格 `unitPrice` 误录入为了 `0`。
  4. 产能规划中，2026-06 存在 1500 片预测需求，但未配置该月份任何工厂的产能。
  5. 营业目标中，2027 年配置了 1200M TWD 的 Forecasts，但 `bpTargets` 中 2027 年的目标额配置为 `0` (或 null)。
  6. SKU 主表中录入了一款采用 `TWD` 计价的商品，但 Parameters 页面中没有配置 TWD/USD 汇率。

---

## 阶段一：Products 属性缺失就地自愈 (Quick Fix Drawer)

1. **操作者**：角色甲 (Owner)。
2. **操作步骤**：
   * 登录系统，切换至 `smoke-test-workspace-v136` 工作区。
   * 进入 **Products (产品 SKU)** 菜单页。
   * 在 SKU Table 列表中，定位到 `SKU-DIRTY-01` 行。首列显眼地标有红色的 `<ExclamationCircleOutlined />` 错误图标。
   * 鼠标悬浮 (Hover) 在图标上，Tooltip 提示：“产品 SKU 缺失生产属性: layerCount”。
   * **核心动作**：点击该红色错误图标。
3. **预期结果**：
   * 页面没有发生跳转，屏幕右侧平滑拉出 `SKU Quick Fix Drawer`。
   * 抽屉标题显示 `快速修复产品 SKU - SKU-DIRTY-01`。
   * 表单中，`层数 (layerCount)` 输入框的边框呈醒目的红色高亮，并伴有“此为核心分析属性，不能为空”的提示。且焦点已自动定位在此输入框中。
4. **自愈动作**：
   * 在 `layerCount` 输入框中填入 `8`。
   * 点击抽屉下方的“确认自愈 (Fix SKU)”按钮。
5. **预期结果**：
   * 保存按钮进入 Loading 状态，大约 0.5 秒内写入完毕，抽屉平滑关闭。
   * 页面上，`SKU-DIRTY-01` 行的红色 Error 图标即时隐去，化为绿色的 Check。
   * 页面没有任何强制整页刷新的闪烁，体验极度顺滑。

---

## 阶段二：Forecast 孤儿行与 0 价格自愈 (Popover & Spreadsheet)

1. **操作者**：角色甲 (Owner)。
2. **操作步骤 (测试孤儿预测 Quick Fix)**：
   * 进入 **Forecasts (需求预测)** 菜单页。
   * 定位到绑定了 `SKU-GHOST-99` 的预测行，该行底色呈淡红色警示，且 SKU ID 旁带有红色警告 Badge。
   * **核心动作**：点击该 Badge。
3. **预期结果**：
   * 弹出 `Orphan Forecast Quick Fix Popover` 气泡。
   * 气泡内显示：“此预测绑定的 SKU [SKU-GHOST-99] 在系统中不存在！请选择修复路径：”
4. **自愈动作**：
   * 在气泡内的 `Select SKU` 下拉框中，展开后选中已存在的合法 SKU `SKU-NORMAL-02`。
   * 点击气泡下方的“重绑定并保存”按钮。
5. **预期结果**：
   * 气泡关闭，该行的淡红底色即时消除，SKU 成功更改为 `SKU-NORMAL-02`，报错隐去。
6. **操作步骤 (测试 0 价格快速修复)**：
   * 在 Forecast 列表中，定位到 2026-03 价格为 `0` 的预测记录单元格。单元格高亮浅黄底色，并带有黄色警告图标。
   * 双击该单元格。
7. **预期结果**：
   * 单元格进入就地 InputNumber 编辑态。
8. **自愈动作**：
   * 键盘输入 `120`，按下 Enter 键。
9. **预期结果**：
   * 数据落盘成功，浅黄警示底色消除，且顶部 ActionBar 的总收入与 BP Target 达成率即时联动更新。

---

## 阶段三：跨页面容量缺失引导与定位高亮 (Guided & Navigation Fix)

1. **操作者**：角色甲 (Owner)。
2. **操作步骤 (测试产能缺失 Guided Fix)**：
   * 进入 **Capacity (产能规划)** 页面。
   * 顶部 ActionBar 下方出现显眼的红色大 Alert：“发现预测需求月份缺少产能规划（如：2026-06），请在该月份添加产能”。
   * **核心动作**：点击 Alert 右侧的“去解决 (Remediation)”按钮。
3. **预期结果**：
   * 弹出 `Capacity Deficiency Remediation Modal` 对话框。
   * 窗口呈现两条清晰路径：“选项一：去配置产能 (推荐)” 与 “选项二：核对该月需求预测”。
4. **自愈动作**：
   * 点击“选项一：去配置产能”链接。
5. **预期结果**：
   * 页面路由成功跳转至 `/capacity` 页面。
   * 页面自动加载了 2026 年，视口自动平滑向下滚动，直接聚焦定位并锚定在 **06 月的 Core Panel** 输入框上。
   * 06 月的 Core Panel 输入框呈现明显的橘黄色虚线高亮并闪烁 3 下，极具视觉冲击力。
6. **自愈动作**：
   * 在该输入框填入产能 `2000`，点击保存。
7. **预期结果**：
   * 保存成功，顶部 Alert 自动消失，计算出的 WPI 瓶颈即时更新。

## 阶段四：参数汇率缺失快捷补齐 (Exchange Rate Popover)

1. **操作者**：角色甲 (Owner)。
2. **操作步骤**：
   * 进入 **Parameters (参数设定)** 页面。
   * 汇率 Card 标题旁显示红色 Error Badge。鼠标悬浮提示：“由于存在以 TWD 计价的商品，必须配置该币别的汇率”。
   * **核心动作**：点击该 Error Badge。
3. **预期结果**：
   * 原地弹出 `Exchange Rate Quick Fix Popover` 气泡。
   * 气泡内自动算出：“当前有 2 个 SKU 采用 TWD 计价，但尚未配置汇率。请输入 TWD/USD 常数汇率以完成自愈：”
4. **自愈动作**：
   * 在输入框内填入 `0.031`，点击“即时补全并保存”。
5. **预期结果**：
   * 气泡关闭，红色 Badge 即时变绿（显示 Check 图标）。
   * 切换至 Dashboard 页面，所有 TWD SKU 折算出的营业收入获得正确计算，不再显示 NaN。

## 阶段五：Viewer 角色只读防线硬性拦截

1. **操作者**：角色乙 (Viewer)。
2. **操作步骤**：
   * 退出角色甲，以角色乙 (Viewer 只读权限) 登录系统。
   * 重复上述阶段一的步骤：进入 **Products** 页面，找到依然有属性缺失的 SKU 行。
3. **预期结果**：
   * 角色乙作为 Viewer，能**完整且正确地看到**红色的 Error 图标、黄色警告图标，且鼠标悬浮时 Tooltip 的文案提示与 Owner 角色完全一致（保障完整知情权）。
4. **核心防线动作**：
   * 角色乙尝试点击该红色 Error 图标。
5. **预期结果**：
   * **界面无任何弹窗、无任何 Drawer 拉出，保持绝对的静默**。
   * 表格中所有的可编辑单元格、参数输入框均呈现灰色 Disabled 状态，无法做出任何物理修改，完美守护数据防线！
