# v1.35.0 Input Page Data Quality 冒烟测试脚本 (Smoke Test Script)

本脚本用于指导 QA 或开发人员，在 `v1.35.0` 数据质量前移特性发布后，通过浏览器手动注入特定“脏数据”来验证各输入页面的 DQ 警示、图示与 Tooltip 交互。

---

## 測試場景一：Products.tsx (產品 SKU 頁) DQ Visibility 驗證

### 1. 驗證 SKU 屬性不完整 (Error) 與單價為 0 (Warning)
- **操作步驟**：
  1. 使用 **Editor** 帳號登入系統，導航至 `/products`。
  2. 點擊“新增產品”按鈕，在彈出的 Modal 表單中，僅填寫 `SKU 编码` (例如：`TEST-DIRTY`)，而將 `晶片長度`、`晶片寬度`、`層數` 留空或填寫為 `0`；單價填寫為 `0`。點擊保存。
  3. 觀察表格中該 SKU 的行級視覺表現。
- **預期結果**：
  - 在 `TEST-DIRTY` SKU 的第一列（或 SKU 编码旁），醒目地渲染出一個紅色的 `<ExclamationCircleOutlined />` 錯誤圖標。
  - 將鼠標懸浮至該紅色圖標上，彈出 Tooltip 詳細文案，列出缺失的所有生產屬性（如：`晶片長度`, `晶片寬度`, `層數`）。
  - 單價單元格背景呈現淡黃色，並伴隨黃色警告圖標，懸浮提示“單價為 0”。

### 2. 驗證不支持的幣別 (Warning)
- **操作步驟**：
  1. 編輯任意 SKU，將其幣別欄位修改為 `EUR`（歐元）並保存。
  2. 觀察幣別列的視覺表現。
- **預期結果**：
  - 幣別單元格出現黃色警告圖標，懸浮提示：“不支持的幣別: EUR，請使用 USD, TWD 或 CNY”。

## 測試場景二：Forecasts.tsx (需求預測頁) DQ Visibility 驗證

### 1. 驗證孤兒預測高亮 (Error)
- **操作步驟**：
  1. 導航至 `/forecasts`（或預測 Lab 頁）。
  2. 在本地手動製造一筆預測，使其 `skuId` 對應一個系統中已被刪除或不存在的 SKU。
  3. 觀察預測編輯表格。
- **預期結果**：
  - 該孤兒預測整行高亮為淺紅色底色，且 SKU 欄位顯示紅色錯誤圖標。
  - 鼠標懸浮顯示：“預測指向的 SKU 不存在，此預測需求將無法在分析中被正確歸類”。

### 2. 驗證年份預測不完整警告 (Warning)
- **操作步驟**：
  1. 對某個正常 SKU（例如 `SKU-A`），在 `2026` 年僅錄入 `1` 月至 `8` 月共 8 個月的預測需求，其餘 4 個月留空。
  2. 觀察預測頁面的年份切換器與 SKU 卡片。
- **預期結果**：
  - 年份選擇器（或該行開頭）出現黃色 `<InfoCircleOutlined />` 警告標誌。
  - 懸浮提示：“當前年份數據不完整（僅有 8/12 個月預測），這將導致年度分析出現非意圖的斷崖”。

## 測試場景三：CapacityPlan.tsx (產能規劃頁) DQ Visibility 驗證

### 1. 驗證產能配置缺失大 Alert (Error)
- **操作步驟**：
  1. 在預測頁面中，為 `2026-06` 月份錄入一筆有效的預測需求（例如 `100,000 pcs`）。
  2. 導航至 `/capacity`（產能規劃頁）。
  3. 確認此時產能規劃表格中，**沒有**任何工廠配置了 `2026-06` 月份的產能數據（若有，請刪除該月產能）。
  4. 觀察產能規劃頁面頂部。
- **預期結果**：
  - 在頁面頂部 ActionBar 下方，赫然出現一條紅色的 `<Alert className="abf-alert-page" type="error" />` 橫幅。
  - 文案提示：“發現預測需求月份缺少產能規劃：2026-06。這會使該月份的產能分析失真，請在該月份添加產能配置”。

### 2. 驗證 BU 產能嚴重缺失 (Error)
- **操作步驟**：
  1. 確認 `SKU-A` 的 `層數` 大於等於 4（高層數產品，需要 BU 產能）。
  2. 在預測頁面中，為 `SKU-A` 在 `2026-07` 月份錄入有效預測需求。
  3. 導航至 `/capacity` 頁面，找到 `2026-07` 月份的產能配置，將所有工廠的 `buPanelPerDay` 全部修改為 `0`。保存。
  4. 觀察產能表格的 `2026-07` 行。
- **預期結果**：
  - 產能表格在 `2026-07` 月份的 `buPanelPerDay` 單元格上，渲染紅色邊框與 Error 警告圖標。
  - 鼠標懸浮 Tooltip 提示：“該月存在高層數產品需求，但 Build-up 產能配置為 0，這會導致 100% 的產能瓶頸”。

## 測試場景四：BpTargets.tsx 與 Parameters.tsx DQ Visibility 驗證

### 1. 驗證目標與需求脫節 (Warning)
- **操作步驟**：
  1. 導航至 `/bp-targets`。
  2. 在 `2028` 年錄入 `150` (Million TWD) 的營業額目標。
  3. 確認在預測頁面中，`2028` 年沒有錄入任何 SKU 的預測數據。
  4. 觀察 `2028` 年營業目標行。
- **預期結果**：
  - 表格在 `2028` 年行首顯示黃色警告圖標，懸浮提示：“該年份配置了營業目標，但沒有錄入任何預測需求，營業達成率分析將固定為 0.0%”。

### 2. 驗證匯率缺失警告 (Error)
- **操作步驟**：
  1. 將某個 SKU 的幣別修改為 `CNY`。
  2. 導航至 `/parameters`。
  3. 將匯率模式切換為“常數匯率”，並將 `USD to CNY` 匯率輸入框清空或填寫為 `0`。保存。
  4. 觀察 Parameters 頁面。
- **預期結果**：
  - 在匯率設定卡片的匯率輸入框或標題旁，出現紅色 Error 圖標，懸浮提示：“由於存在以 CNY 計價的產品數據，必須配置該幣別的匯率”。

## 測試場景五：多人協作 Viewer 唯讀角色只讀驗證

- **操作步驟**：
  1. 以 **Viewer** 角色帳號登入系統，分別導航至上述五大頁面。
  2. 重複觸發上述各 DQ 異常場景（或查看已存在異常的測試數據）。
- **預期結果**：
  - 所有 Error、Warning、Info 的彩色警告圖標、紅/黃色 Alert 橫幅以及 Tooltip 懸浮文案，**在 Viewer 唯讀狀態下 100% 完整展示**。
  - 對應的數值單元格、Save 保存按鈕、還原預設值按鈕依然保持 **置灰 (disabled)**，無法點擊編輯，防線固若金湯。
